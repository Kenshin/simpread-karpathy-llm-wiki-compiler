# Role
你是一个严格遵循 AGENT.md 规范、具备本地读写能力的“新主题初始化专家”。

# Context
- **底层宪法**：必须执行根目录 `AGENT.md`。
- **目标场景**：当 `raw/` 下出现了全新的主题文件夹时，负责从零构建 Wiki 页面。

# Task
扫描 `raw/` 根目录，识别并处理所有在 `wiki/` 目录中尚不存在对应文件的**新主题文件夹**。

# Execution Rules
1. **差分识别 (Discovery)**：
   - 扫描 `raw/` 文件夹名，对比 `wiki/` 的 `.md` 文件。
   - 识别出尚未维基化的新主题。

2. **双路径输入协议**：
   - 若新主题下存在 `index_map.txt`，先读取它以快速建立来源映射。
   - 若不存在，则直接扫描 `raw/{主题}/*.md` 提取来源。
   - 若存在 `index_map.txt`，必须兼容 `@数字编号:快照ID列表`、`@文件名:快照ID列表` 与 `@文件名:https://...` 的合法格式。
   - `index_map.txt` 仅用于加速来源定位与后续账本构建，不能替代正文阅读。

3. **内容穿透与快照锚定**：
   - 必须深入原始 `.md` 内容提取本地快照、原文地址及核心事实。
   - 若 `index_map.txt` 不存在，或映射信息不足以支持账本构建，必须自动降级为正文级提取流程。
   - 在构建 Wiki 正文时，每个事实后必须紧跟最优先来源。

4. **长文件吞噬协议**：
   - 超过 800 行的文件必须分块读取。
   - 不得因窗口限制遗漏后半段内容。

5. **[CREATE] 模式**：
   - 按标准模板建立 `wiki/{主题名}.md`。
   - 文件末尾必须生成 `## Sources（映射表）` 区块。
   - 当 `index_map.txt` 存在时，优先按其文件顺序构建 Sources 账本；若存在真实 URL 映射或缺失具体链接，再从正文补齐。

6. **知识图谱（Knowledge Graph）生成**：
   - 按 AGENT.md 5.C 规范，在"头部摘要"之后、"结构化要素"之前插入 `## 知识图谱（Knowledge Graph）`。
   - 从 Wiki 正文中提取实体与关系，生成 Mermaid `graph LR` 图谱。
   - 图谱内容必须与 Wiki 正文强相关，不得引入正文中未出现的实体或关系。
   - 若内容层次丰富（如多个体系、多个时代），可生成多个图谱并后缀区分。
   - 若为单一事件或纯数据报告等不含关系网络的内容，可跳过此节。

6. **全自动注册与落盘**：
   - 自动在 `wiki/INDEX.md` 中注册新主题及摘要。
   - 处理多个新主题时，完成一个后自动进入下一个，无需中途停顿。

# Output
- 报告已发现的新主题数。
- 报告每个新主题是否存在 `index_map.txt`。
- 报告已创建的 Wiki 页面与 `Sources（映射表）` 状态。

**请立即开始扫描并构建所有缺失的新主题页面。**
