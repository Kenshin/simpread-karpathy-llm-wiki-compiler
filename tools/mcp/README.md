# SimpRead MCP Server API 文档

## 📋 概述

SimpRead MCP Server 提供了一组工具，用于查询和检索简悦（SimpRead）本地快照数据。

**服务器信息：**
- 名称：`SimpRead MCP Server`
- 版本：`2.0.0`
- 协议：Model Context Protocol (MCP)

---

## 🛠️ 工具列表

### 1. help

获取 SimpRead MCP 的帮助信息。

**工具名称：** `help`

**参数：**

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `type` | string | ✅ | 固定值 `"help"` |

**示例调用：**
```json
简悦阅读助手帮助
```

**返回值：**
- 类型：`text`
- 内容：帮助文档文本

---

### 2. get_daily

按日期范围获取简悦未读内容列表。

**工具名称：** `get_daily`

**参数：**

| 参数名 | 类型 | 必填 | 描述 | 可选值 |
|--------|------|------|------|--------|
| `type` | string | ✅ | 时间范围类型 | `today`, `yesterday`, `daily`, `week`, `last7`, `last30` |

**参数详解：**

- `today` - 今天收藏的内容
- `yesterday` - 昨天收藏的内容
- `daily` - 今日内容（通常包含昨天）
- `week` - 本周收藏的内容
- `last7` - 最近 7 天收藏的内容
- `last30` - 最近 30 天收藏的内容

**示例调用：**

```
今日阅读回顾
→ get_daily(type="today") 以简报形式返回今日阅读内容

获取最近一周的未读文章
→ get_daily(type="last7") 以简报形式返回最近一周的阅读回顾
```

**返回值：**
- 类型：`text`
- 内容：包含文章标题和本地快照链接的列表

**返回格式示例：**

```
# 曾经全球第一的日本机器人，迎来「黄昏时刻」-36 氪
本地快照 http://localhost:7026/reading/5219
原文地址 https://36kr.com/p/3809192363859970
日期 2026年05月15日 11:23:43
标签 观察/商业历史
请调用 get_snapshot 并将 5219 作为参数传入

# 美联储最具争议主席诞生，56 岁沃什险胜当选，妻子是雅诗兰黛女继承人，岳父是特朗普多年好友 - 36 氪
本地快照 http://localhost:7026/reading/5218
原文地址 https://36kr.com/p/3809919886581508
日期 2026年05月15日 11:22:00
标签 观察/商业历史
请调用 get_snapshot 并将 5218 作为参数传入
```

### 3. search_content

按关键词全文搜索简悦内容。

**工具名称：** `search_content`

**参数：**

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `str` | string | ✅ | 搜索关键词 |

**示例调用：**

```
请查询关键词 OpenAI
→ search_content(str="OpenAI") 以简报形式返回 OpenAI 相关的内容

查询关键词 openai 在此结果检索 马斯克 内容，并生成简报 ~r
→ search_content(str="openai") 在检索结果中继续检索关键词 `马斯克` 以简报 + 原文链接形式返回马斯克相关内容的简报
```

**返回值：**
- 类型：`text`
- 内容：取决于搜索结果数量（由 `MIN_PAGING` 环境变量控制，默认 30）

### 4. search_tag

按标签筛选内容。

**工具名称：** `search_tag`

**参数：**

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `tag` | string | ✅ | 标签名称 |

**示例调用：**

```
获取标签为 AI 的文章，并生成简报
→ search_tag(tag="AI") 以简报形式返回 AI 相关的内容

查找"科技史话"标签下的内容 ~r
→ search_tag(tag="科技史话") 以简报 + 原文链接形式返回科技史话相关的内容。
```

**返回值：**
- 类型：`text`
- 内容：取决于搜索结果数量（逻辑与 `search_content` 相同）

**返回值类型 A：结果数量 < MIN_PAGING（直接返回文章列表）**

**返回值类型 B：结果数量 >= MIN_PAGING（返回分页指示）**

返回格式与 `search_content` 相同，包含分页指示和完整的 Output Format 规则。

**注意事项：**
- 标签名称支持层级结构，如 `科技史话/AI战争`
- 支持模糊匹配，使用 Fuse.js 搜索引擎
- 搜索逻辑：支持精确匹配（`="科技史话"`）和前缀匹配（`^"科技史话/"`）

### 5. get_unread_list

分页获取简悦未读列表。通常在 `search_content` 或 `search_tag` 返回分页指示后调用。

一般不会主动用到此方法，而是通过调用 `search_content` 或 `search_tag` 得到文章列表中后由 AI 调用此方法获取文章的详细信息。

**工具名称：** `get_unread_list`

**参数：**

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `str` | string | ✅ | 搜索内容（与调用 `search_content` 时的 `str` 一致） |
| `page` | number | ✅ | 页码（从 0 开始） |

**示例调用：**
```json
{
  "name": "get_unread_list",
  "arguments": {
    "str": "星巴克",
    "page": 0
  }
}
```

```json
{
  "name": "get_unread_list",
  "arguments": {
    "str": "星巴克",
    "page": 1
  }
}
```

**返回值：**
- 类型：`text`
- 内容：分页后的文章列表

**返回格式示例：**

```text
# Background

检索结果共计 20 条，共计 160 行，请确保完整读取全部内容。

# 查询结果

# 星巴克，终于活成了中国人想要的样子？-36 氪
本地快照 http://localhost:7026/reading/4979
原文地址 https://36kr.com/p/3760408941871873
日期 2026年04月10日 14:56:15
标签 星巴克
请调用 get_snapshot 并将 4979 作为参数传入

# 引入资本后的星巴克中国：不提降价，开更多店型
本地快照 http://localhost:7026/reading/4971
原文地址 https://www.jiemian.com/article/14224574.html
日期 2026年04月09日 11:38:07
标签 星巴克
请调用 get_snapshot 并将 4971 作为参数传入

```

**分页逻辑：**
- 每页显示数量由 `STEP` 环境变量控制（默认 20）
- `page` 从 0 开始
- 返回内容包含文章列表和输出格式规则

### 6. get_unread_by_idx

通过快照 ID 获取特定文章的详细信息（元数据）。

一般不会主动用到此方法，而是通过调用 `search_content` 或 `search_tag` 得到文章列表中后由 AI 调用此方法获取文章的详细信息。

**工具名称：** `get_unread_by_idx`

**参数：**

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `idx` | string | ✅ | 快照 ID（数字字符串） |

**示例调用：**
```json
{
  "name": "get_unread_by_idx",
  "arguments": {
    "idx": "5017"
  }
}
```

**返回值：**
- 类型：`text`
- 内容：文章的元数据信息

**返回格式示例：**

```text
# 稍后读 5017 详细信息

**ID**：5017
**标题**：英伟达市值升破 5.26 万亿美元，时隔半年再度刷新纪录
**本地快照**：http://localhost:7026/reading/5017
**原文地址**：https://36kr.com/p/3785831527767047
**建立时间**：2026年04月28日 11:27:10
请根据以上信息，结合本地快照的内容，进行详细分析和总结。
```

**使用场景：**
- 获取文章的原始 URL（用于 `-r` 参数替换本地快照链接）
- 获取文章的完整元数据
- 验证快照 ID 是否有效

**与 `get_snapshot` 的区别：**

| 工具 | 返回内容 | 使用场景 |
|------|----------|----------|
| `get_unread_by_idx` | 文章元数据（标题、URL、时间、ID） | 获取文章信息和原始链接 |
| `get_snapshot` | 文章完整正文（Markdown） | 阅读和分析文章内容 |

### 7. get_snapshot

获取本地快照的完整正文内容。

一般不会主动用到此方法，而是通过调用 `search_content` 或 `search_tag` 得到文章列表中后由 AI 调用此方法获取文章的详细信息。

**工具名称：** `get_snapshot`

**参数：**

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `str` | string | ✅ | 快照 ID（数字字符串） |

**示例调用：**
```json
{
  "name": "get_snapshot",
  "arguments": {
    "str": "5017"
  }
}
```

**返回值：**
- 类型：`text`
- 内容：文章的完整正文内容（Markdown 格式）

**返回格式示例：**

```markdown
英伟达市值升破 5.26 万亿美元，时隔半年再度刷新纪录

当地时间 4 月 27 日，芯片巨头英伟达股价单日上涨 4%，收盘价定格在 216.61 美元的历史高位...

当前美股市值前三甲的格局锁定在英伟达、谷歌、苹果三家科技公司上。英伟达以 5.26 万亿美元的市值遥遥领先...

[图片]

回溯 2024 年末的排名，苹果、英伟达、微软分别以 3.79 万亿美元、3.29 万亿美元、3.13 万亿美元占据前三席位...
```

**实现细节：**
- 从 `http://localhost:7026/reading/{idx}` 获取 HTML 内容
- 提取 `<sr-rd-content>` 标签内的正文
- 使用 Turndown 将 HTML 转换为 Markdown（ATX 风格标题）

**使用场景：**
- 阅读文章完整内容
- 基于文章内容生成分析简报
- 提取文章中的具体数据和引用

**与 `get_unread_by_idx` 的区别：**

| 工具 | 返回内容 | 使用场景 |
|------|----------|----------|
| `get_unread_by_idx` | 文章元数据（标题、URL、时间等） | 获取文章信息和原始链接 |
| `get_snapshot` | 文章完整正文（Markdown） | 阅读和分析文章内容 |

## 🔄 典型工作流

### 工作流 1：搜索并阅读（少量结果）

```
1. search_content(str="星巴克")
   → 返回 15 条结果（< MIN_PAGING）
   → 直接返回文章列表

2. get_snapshot(str="4979")
   → 获取 ID 为 4979 的文章完整内容
```

### 工作流 2：搜索并阅读（大量结果，需要分页）

```
1. search_content(str="星巴克")
   → 返回 153 条结果（>= MIN_PAGING）
   → 返回分页指示：调用 get_unread_list 8 次

2. get_unread_list(str="星巴克", page=0)
   → 获取第 1 页的 20 条文章标题

3. get_unread_list(str="星巴克", page=1)
   → 获取第 2 页的 20 条文章标题

4. ... 重复调用直到所有分页获取完毕

5. 从所有标题中筛选相关内容
   → 例如筛选"新任 CEO"相关的文章

6. get_snapshot(str="3189")
   → 只获取筛选后的文章内容
```

### 工作流 3：获取原始链接（用于 -r 参数）

```
1. search_tag(tag="科技史话/AI战争")
   → 获取标签下的文章列表

2. get_snapshot(str="5017")
   → 获取文章内容，引用本地快照链接

3. get_unread_by_idx(idx="5017")
   → 获取文章原始 URL

4. 将本地快照链接替换为原始 URL
   → http://localhost:7026/reading/5017 → https://36kr.com/p/...
```

### 工作流 4：近期回顾

```
1. get_daily(type="last7")
   → 获取最近 7 天的文章列表

2. get_snapshot(str="5017")
   → 获取感兴趣的文章内容

3. get_snapshot(str="5016")
   → 继续获取其他文章
```

## 📝 返回数据格式

由 `search_content`、`search_tag` 当 **结果数量 >= MIN_PAGING（分页指示）** 返回：

### 检索分页显示（paging）

```text
共搜索到 153 条相关内容，已分为 8 页，每页 20 条，请按照下面的依次读取分页内容：

请调用 get_unread_list 并传入当前页数 0 来获取第 1 页的内容。

请调用 get_unread_list 并传入当前页数 1 来获取第 2 页的内容。

请调用 get_unread_list 并传入当前页数 2 来获取第 3 页的内容。
```

### 文章列表格式（convertUnreadsLite）

由 `search_content`、`search_tag`、`get_daily`、`get_unread_list` 当 **结果数量 < MIN_PAGING（分页指示）** 返回：

```text
# {标题}
本地快照 http://localhost:7026/reading/{idx}
原文地址 {url}
日期 {create}
标签 {tags}
请调用 get_snapshot 并将 {idx} 作为参数传入
```

**字段说明：**

| 字段 | 说明 | 示例 |
|------|------|------|
| `标题` | 文章标题 | `星巴克，终于活成了中国人想要的样子？-36 氪` |
| `idx` | 快照 ID（数字） | `4979` |
| `url` | 原始文章 URL | `https://36kr.com/p/3760408941871873` |
| `create` | 收藏时间 | `2026年04月10日 14:56:15` |
| `tags` | 标签列表（逗号分隔） | `星巴克` 或 `科技史话/AI战争, 科技史话/芯片战争` |

### 文章元数据格式（get_unread_by_idx）

```text
# 稍后读 {idx} 详细信息

**ID**：{idx}
**标题**：{title}
**本地快照**：http://localhost:7026/reading/{idx}
**原文地址**：{url}
**建立时间**：{create}
请根据以上信息，结合本地快照的内容，进行详细分析和总结。
```

### 文章内容格式（get_snapshot）

返回文章的 Markdown 格式正文内容，由 HTML 转换而来：

- 标题使用 ATX 风格（`#`、`##`）
- 图片保留原始链接
- 代码块和引用保持原样

## ⚙️ 环境变量配置

MCP 服务器支持以下环境变量：

| 变量名 | 默认值 | 描述 |
|--------|--------|------|
| `PROMPT` | `"default"` | 输出提示词模板 |
| `MIN_PAGING` | `30` | 触发分页的最小结果数量 |
| `STEP` | `20` | 每页显示的文章数量 |

**PROMPT 可选值：**
- `default` - 默认输出格式（包含 Workflow + Output Format）
- `omni` - 全能模式输出格式（更详细的分析结构）
- `自定义字符串` - 自定义提示词模板

**MIN_PAGING 说明：**
- 当 `search_content` 或 `search_tag` 的结果数量 >= `MIN_PAGING` 时，返回分页指示
- 当结果数量 < `MIN_PAGING` 时，直接返回文章列表
- 不建议修改，如果 LLM 可读上下文较多时，可适当增加此值

**STEP 说明：**
- 控制 `get_unread_list` 每页返回的文章数量
- 分页总数 = ceil(结果数量 / STEP)
- 不建议修改，如果 LLM 可读上下文较多时，可适当增加此值

## 🐛 错误处理

### 常见错误

**错误 1：无法连接到简悦服务**
```
Error: connect ECONNREFUSED 127.0.0.1:7026
```
**解决方案：** 确保简悦同步助手正在运行

**错误 2：找不到快照**
```
Error: Snapshot not found
```
**解决方案：** 检查快照 ID 是否正确，确认文章已保存本地快照

**错误 3：搜索结果为空**
```
未找到匹配的内容
```
**解决方案：** 
- 检查关键词拼写
- 确认 `simpread_config.json` 中存在相关内容
- 尝试使用标签搜索
